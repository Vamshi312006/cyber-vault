# Purpose

This document defines the execution methodology used throughout this career journey.

Its purpose is to transform long-term goals into consistent daily progress through a structured hierarchy of missions, sprints, modules, and tasks.

Execution is the bridge between planning and achievement.

---

---

# Mission

A mission represents a major objective.

Examples

- Complete Healthcare V2
- Complete Detection Engineering Platform
- Complete Cyber Act Stage 1

A mission may span multiple months.

---

# Sprint

A sprint is a focused execution cycle.

Each sprint should have:

- One primary objective
- Clear deliverables
- Defined completion criteria

A sprint should produce visible progress.

---

# Module

Modules divide a sprint into logical engineering units.

Examples

- Authentication
- RBAC
- Secure File Upload
- Sysmon Parsing
- Sigma Integration

Every module should produce working functionality.

---

# Task

Tasks are the smallest executable units.

A task should generally be completable in one focused work session.

Examples

- Implement login validation
- Add rate limiting
- Write Sigma parser
- Create authentication tests

---

# Execution Cycle

Every task follows the same cycle.

↓

↓

↓

Test

↓

↓

↓

Review

Skipping stages should be avoided.

---

# Daily Workflow

Every work session begins by checking:

- CURRENT_STATUS.md
- Current Sprint
- Current Module
- Current Task

At the end of each session:

- Update CURRENT_STATUS.md
- Record progress
- Identify next task

Avoid switching between unrelated work unless blocked.

---

# Context Switching

Minimize unnecessary context switching.

During a sprint:

- Avoid starting unrelated features.
- Avoid redesigning completed architecture.
- Avoid researching technologies that are not immediately required.

Maintain focus until the sprint objective is complete.

---

# Definition of Done

A task is complete only when:

✓ Implementation works

✓ Tests pass

✓ Documentation updated

✓ Git committed

✓ Progress tracker updated

If applicable:

✓ Resume evidence recorded

---

# Blockers

When blocked:

1. Identify the blocker.
2. Research only what is necessary.
3. Resolve the issue.
4. Continue execution.

Do not replace execution with endless research.

---

# Review

Every sprint ends with a review.

Review includes:

- Completed work
- Remaining work
- Lessons learned
- Technical debt
- Resume updates
- Portfolio updates
- Next sprint planning

---

# Final Principle

Consistent execution creates momentum.

Momentum creates expertise.

Expertise creates opportunities.

The objective is not to stay busy.

The objective is to consistently complete meaningful work.
