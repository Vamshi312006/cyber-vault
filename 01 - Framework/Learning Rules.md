# Learning Rules

## Purpose

Every technical topic in this vault follows the same learning pipeline. The goal is to first understand **what** a technology does, then observe it, then understand **how** it works internally, and finally learn how to attack, defend, and document it.

---

# Learning Pipeline

## Phase 1 — Functional Model

### Questions to Answer

- What is it?
- What problem does it solve?
- How does it help?
- What are its components?
- Where is it used?
- What are its limitations?

### Goal

Understand the purpose and practical value without learning the internals.

---

## Phase 2 — Practical Observation

### Activities

- Observe it in action.
- Configure it.
- Capture packets, logs, or events.
- Identify inputs and outputs.
- Break it if possible.
- Record observations.

### Goal

Build intuition before learning the implementation.

---

## Phase 3 — Mechanism

### Questions to Answer

- How does it work internally?
- What is the step-by-step workflow?
- Which protocols or components participate?
- Why was it designed this way?
- What happens when something fails?

### Goal

Understand the implementation and reasoning behind the technology.

---

## Phase 4 — Practical Reinforcement

### Activities

- Repeat the lab.
- Explain every step using the mechanism.
- Verify that observations match the theory.

### Goal

Connect internal understanding with real behavior.

---

## Phase 5 — Security Perspective

For every technology identify:

- Common attacks
- Common misconfigurations
- Indicators of compromise
- Detection opportunities
- Mitigations
- SOC / Blue Team relevance

### Goal

Understand how the technology is abused and defended.

---

## Phase 6 — Documentation

Every topic should contain:

- What is it?
- Problem it solves
- Benefits
- Components
- Uses
- Limitations
- Practical Observation
- Mechanism
- Security
- Detection
- Commands / Labs

---

# Completion Criteria

A topic is complete only when:

- Functional questions are answered.
- Practical observation is completed.
- Mechanism is understood.
- Practical reinforcement is performed.
- Security perspective is covered.
- Notes are documented in the standard format.
