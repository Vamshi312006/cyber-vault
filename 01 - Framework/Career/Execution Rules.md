# Execution Rules

## Rule 1

Analyze enough.

Then commit.

---

## Rule 2

Every topic learned must produce an artifact.

---

## Rule 3

Don't ask

"What should I learn?"

Ask

"What evidence will this create?"

---

## Rule 4

One flagship project.

Many releases.

Never abandon it.

---

## Rule 5

One effort should produce multiple hiring signals.

Example

Feature

↓

GitHub

↓

Article

↓

Hackathon

↓

Resume

↓

Portfolio

↓

Interview discussion

---

## Rule 6

Ship Version 0.1.

Improve later.

Don't wait for perfection.

---

## Rule 7

Execution is measured by shipped artifacts, not hours worked.
